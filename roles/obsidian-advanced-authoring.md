# Obsidian Advanced Authoring Guide

## Objective
Produce wiki pages that fully leverage Obsidian features and improve graph readability, navigation, and long-term maintenance.

## Internal Linking
- Use `[[Page Name]]` for conceptual neighbors
- Use aliases when needed: `[[Canonical Page|Readable Label]]`
- Avoid over-linking every sentence; prioritize high-signal entities

## Graph-aware Writing
- Introduce named entities consistently to strengthen graph clustering
- Keep canonical naming stable across entries
- Add a focused related-entry section instead of random in-line links

## Structural Conventions
- Keep one H1 and multiple meaningful H2 sections
- Use concise sections with clear scope boundaries
- Add `## References` for traceability

## Backlink Value
- Mention adjacent topics explicitly to improve backlink relevance
- Prefer reciprocal links between tightly coupled entries

## Transclusion-friendly Blocks
- Write short, standalone paragraphs that can be embedded
- Avoid references like “as mentioned above” that break when transcluded
