# Open Source

## Concept Definition
Open source refers to a development model and licensing philosophy for software where the source code is made publicly available under a license that grants users the rights to use, study, modify, and distribute the software and its source code to anyone and for any purpose. The formal, canonical definition is maintained by the [[Open Source Initiative (OSI)]] in the [Open Source Definition](https://opensource.org/osd), which outlines ten criteria a license must meet.

## Scope and Boundaries
The term "open source" is most precisely applied to software, though its principles have influenced other fields (open content, open hardware). Its core scope is defined by licensing and collaborative practice. It is distinct from, though historically related to, the "free software" movement, which emphasizes user freedoms as an ethical imperative. Open source is also distinct from public domain software (which has no copyright) and source-available software (where code is viewable but not necessarily modifiable or redistributable under open terms).

## Core Principles
The principles are codified in the Open Source Definition. Key tenets include:
1.  **Free Redistribution:** The license cannot restrict anyone from selling or giving away the software.
2.  **Source Code:** The program must include source code and allow distribution in source code form.
3.  **Derived Works:** The license must allow modifications and derived works, and must allow them to be distributed under the same terms as the original software.
4.  **No Discrimination:** The license must not discriminate against any person or group or field of endeavor.

## History and Evolution
The collaborative sharing of software code has early precedents in academic and hacker communities. The modern movement was formalized in 1998. Following the release of the Netscape browser source code, a group including Bruce Perens and Eric S. Raymond founded the Open Source Initiative to promote the term "open source" as a pragmatic alternative to "free software" and to create a clear, business-friendly definition. The OSI published the Open Source Definition, adapted from the Debian Free Software Guidelines, to provide a standard for evaluating licenses.

## Common Licenses
Open source licenses govern how software can be used, modified, and shared. They fall into two broad categories:

*   **Permissive Licenses:** Impose minimal restrictions, mainly requiring attribution. They allow code to be incorporated into proprietary, closed-source software.
    *   **MIT License:** A very short, simple license requiring only that the original copyright and license notice be preserved.
    *   **Apache License 2.0:** A more detailed permissive license that includes an express grant of patent rights from contributors and provides protection against patent litigation.

*   **Copyleft Licenses (or Protective Licenses):** Require that derivative works and, in some cases, software that links to the code, be distributed under the same license terms, thus preserving its open source status.
    *   **GNU General Public License (GPL):** The most prominent copyleft license. The GPL ensures that all derivative works remain free software. It exists in versions 2 and 3, with v3 addressing software patents and digital rights management (DRM).

Standardized short identifiers for these licenses (e.g., `MIT`, `Apache-2.0`, `GPL-3.0`) are maintained by the SPDX specification.

## Governance Models
How open source projects are managed varies significantly:

*   **Foundation-Led:** A non-profit foundation (e.g., The [[Linux Foundation]], Apache Software Foundation, Eclipse Foundation) provides a neutral governance structure, legal support, and funding. Examples include the Linux kernel (Linux Foundation) and Apache HTTP Server (ASF). This model is common for large, strategic projects.
*   **Community-Driven (or "Benevolent Dictator For Life" - BDFL):** A single individual or a small core team retains final decision-making authority, though contributions are accepted from a broad community. Examples include the Python programming language (historically) and many projects on [[GitHub]].
*   **Corporate-Sponsored:** A single company employs the core developers and drives the project's roadmap, though it may accept external contributions. Examples include React (Meta) and Android (Google).

## Typical Applications
Open source is foundational to modern technology. It powers the majority of web servers (Apache, nginx), forms the core of mobile operating systems (Android), underpins cloud infrastructure (Linux, Kubernetes), and is integral to developer tools (Git, VSCode). Its model is also applied beyond software in open data sets, open educational resources, and open hardware designs.

## Misconceptions and Clarifications
*   **"Open source means free of charge."** While often free in price (gratis), the term's primary meaning is freedom (libre) to use and modify. Some open source software is sold commercially.
*   **"You can do anything with open source code."** Usage rights are defined by the specific license. All licenses have conditions, such as attribution (MIT) or share-alike (GPL).
*   **"Open source projects are unorganized."** Successful projects often have rigorous contribution guidelines, code review processes, and formal governance, especially as they scale.
